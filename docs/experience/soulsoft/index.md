## 주요 정보통신기반시설 인프라 보안 취약점 진단

### 프로젝트 개요

| 항목        | 내용                                                             |
| --------- | -------------------------------------------------------------- |
| **기간**    | 2020.09 ~ 2020.12                                              |
| **업무**    | 공공기관 정보보호 가이드라인을 기준으로 서버 환경의 보안 취약점을 분석 및 보안 강화를 위한 개선 방안을 제시 |
| **대상 환경** | Linux (RedHat, CentOS), Windows Server, UNIX                         |

### 핵심 성과 및 의사결정 이력

#### 1. 취약점 자동 점검 쉘 스크립트 구축을 통한 진단 휴먼 에러 0% 달성
* **문제**
  - 공공기관 대상 주요 정보통신 시스템 인프라는 강력한 법적 보안 조치 및 진단 기준 준수 필요
  - 수동 보안 설정 점검 시 휴먼 에러로 인한 점검 항목 누락 및 리소스 낭비 리스크 상존
* **해결**
  - **진단 스크립트화**: 행정안전부 및 KISA의 "주요정보통신기반시설 기술적 취약점 분석·평가 가이드"에 정의된 수십 가지 점검 기준을 반영한 **자동화 쉘 스크립트(Shell Script)** 자체 작성
  - **점검 범위**: Linux/Windows OS의 권한 정책 설정, 미사용 취약 포트(Telnet/FTP) 스캔, 계정 패스워드 강도 정책 검증 수행
* **성과**
  - **진단 효율성 극대화**: 수십 대의 서버 진단 시간을 단축하고 점검 휴먼 에러 발생율 **0% 달성**
  - **인프라 보안 수준 향상**: 점검 결과를 기반으로 조치 및 수정 보고서를 도출하여 실무 서버 인프라 취약점 보완 완료

## 헬스케어 빅데이터 수집 및 전처리 자동화 스크립트 구축

### 프로젝트 개요

| 구분        | 내용                                                   |
| --------- | ---------------------------------------------------- |
| **기간**    | 2020.09 ~ 2020.12                                    |
| **업무**    | 헬스케어 데이터 수집을 위한 인스타그램 인플루언서 게시글 및 댓글 데이터 수집 웹 크롤러 개발 |
| **기술 스택** | Python, BeautifulSoup, Selenium, Google API                      |

### 핵심 성과 및 의사결정 이력

#### 2. 가상 브라우저 & BeautifulSoup 하이브리드 파싱으로 헬스케어 수집 파이프라인 최적화
* **문제**
  - 소셜 미디어 플랫폼 특유의 로그인 세션 및 비로그인 접근 차단 방화벽 정책 우회 필요
  - 동적 무한 스크롤 형태로 피드가 로드되어 기존의 단순 정적 HTTP Request 라이브러리(requests) 활용 불가
* **해결**
  - **가상 브라우저 자동화**: 세션 인증 흐름을 유지하고 스크롤 다운 트리거 이벤트를 수동 모방하기 위해 **Selenium WebDriver** 구동 엔진 설계
  - **하이브리드 파싱 최적화**: Selenium 단독 브라우저 렌더링에 따른 심각한 CPU/메모리 부하를 줄이기 위해, 브라우저가 읽어온 동적 HTML 문자열을 메모리로 즉시 가로채 **BeautifulSoup**으로 고속 정적 텍스트 파싱을 동시 병행 처리하는 하이브리드 수집 구조 고안
* **성과**
  - **연구 데이터 자산 확보**: 대량의 소셜 피드 게시글 및 댓글 텍스트 정제, 해시태그 파싱을 자동화하여 자연어 분석 분석을 위한 대형 데이터셋 구축 기여
