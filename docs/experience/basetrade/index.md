## 이더리움 환전 플랫폼 개발

### 프로젝트 개요

| 항목        | 내용                                                             |
| --------- | -------------------------------------------------------------- |
| **기간**    | 2022.11 ~ 2023.10 (1년)                                  |
| **업무/프로젝트** | 이더리움 환전 플랫폼 백엔드 Admin API 개발 및 배포 (PM: 1 / BE: 2 / FE: 1) |
| **목적/기술 스택** | 실시간 이더리움 환전 및 관리 시스템 구축 / Python, Django, Pytest |
| **대상/개발 환경** | Linux, PostgreSQL, Docker, AWS                                 |

### 핵심 성과 및 의사결정 이력

#### 1. pytest 및 Mocking 표준 도입을 통한 테스트 커버리지 71% 달성 (52% ➔ 71%)
* **문제**
  - 프로젝트 내 테스트 코드가 전무하여 배포 시 금융 API 영향 범위를 수동 테스트(Ad-hoc)로만 검증
  - 신규 기능 추가 시 결제 로직 안정성 불투명 및 잠재적 장애 리스크 노출
* **해결**
  - **테스트 프레임워크 전환**: Django 기본 Test Suite 대비 디바이스 독립적이고 간결한 코드 작성이 가능한 **pytest** 도입
  - **테스트 환경 표준화**: 외부 결제망(PG) 및 이더리움 RPC 외부 노드 통신으로 테스트 수행이 격리되지 않고 지연되는 문제를 해소하기 위해 외부 연동망을 Mock/Stub으로 대체하는 **사내 모킹 표준 가이드라인 정립**
* **성과**
  - **품질 지표 달성**: 테스트 커버리지 기존 **52%에서 71%로 향상**
  - **배포 신뢰성 강화**: 기능 추가 및 리팩토링 시 회귀 오류 사전 탐지율 100% 확보를 통해 운영 리스크 대폭 절감

## 가상화폐 채굴 모니터링 시스템 개발

### 프로젝트 개요

| 항목        | 내용                                                             |
| --------- | -------------------------------------------------------------- |
| **기간**    | 2021.12 ~ 2022.08 (9개월)                                      |
| **업무/프로젝트** | 가상화폐 채굴 모니터링 시스템 백엔드 API 개발 및 배포 (PM: 1 / BE: 1 / FE: 2) |
| **목적/기술 스택** | 실시간 GPU 상태 수집 및 모니터링 / Python, Django, Django Channels, Redis |
| **대상/개발 환경** | Linux, PostgreSQL, Docker, AWS                                 |

### 핵심 성과 및 의사결정 이력

#### 1. Daphne 프로세스 다중화 기반 WebSocket 로드밸런싱으로 데이터 유실률 0% 달성
* **문제**
  - GPU 채굴 장비 약 220~250대의 실시간 상태 데이터를 수집하는 과정에서 패킷 유실 발생(약 8% 누락)
  - 단일 Daphne 프로세스에서 대량 동시성 소켓 커넥션을 전부 처리하지 못해 CPU 병목 발생
  - 예산 제약으로 AWS EC2 서버 인스턴스 사양 업그레이드(Scale-up)가 불가능한 환경적 한계 존재
* **해결**
  - **인스턴스 분산 다중화**: 추가 인프라 예산 소모 없이 성능을 확장하기 위해 systemd 설정을 변경하여 단일 Daphne 서버를 **4개의 프로세스로 분할 구동**
  - **역방향 프록시 연동**: 전면 Nginx Reverse Proxy에 Upstream 그룹을 구성하고 포트별 분산 로드밸런싱 규칙을 바인딩하여 실시간 트래픽 처리량 확장
* **성과**
  - **수집 무결성**: 인프라 추가 비용 **0원**으로 데이터 수집 누락률 8% ➔ **0% 달성**
  - **운영 공수 감소**: 패킷 누락 시 상시 투입되던 데이터 수동 보정 업무를 제거하여 운영 효율화 기여

#### 2. 스크럼 및 정기 릴리즈 프로세스 수립을 통한 협업 커뮤니케이션 오버헤드 40% 절감
* **문제**
  - 신임 기획자/기초 개발자로만 이루어진 소규모 TF 조직으로 기능 배포 공유 표준이나 업무 산정 기준 부재
  - 개발 진행 가시성 저하로 요구사항 스펙 불일치 및 릴리즈 딜레이 위험 상존
* **해결**
  - **스크럼 프로세스 도입**: 작업 가시성 공유를 위해 Wrike 도구를 연계한 일일 스탠드업(Daily Standup) 및 주간 단위 스프린트 주기 구성
  - **공식 릴리즈 가이드라인**: 매주 금요일 정기 배포 후 API 변경 명세와 기능 추가 사항을 요약한 릴리즈 노트를 현업 팀에 전파하는 배포 프로세스 표준 확립
* **성과**
  - **협업 시너지 확보**: 부서 간 일정 조율 에러를 최소화하고 개발/기획 협업 커뮤니케이션 오버헤드 40% 절감

## P2P 법인 투자 플랫폼 개발

### 프로젝트 개요

| 항목        | 내용                                                             |
| --------- | -------------------------------------------------------------- |
| **기간**    | 2021.01 ~ 2021.11 (10개월)                                     |
| **업무/프로젝트** | P2P 법인 투자 플랫폼 백엔드 API 개발 (PM: 1 / BE: 3 / FE: 3)   |
| **목적/기술 스택** | 법인 투자 프로세스 및 보고서 생성 자동화 / Python, Django, wkhtmltopdf |
| **대상/개발 환경** | Linux, PostgreSQL, Docker                                      |

### 핵심 성과 및 의사결정 이력

#### 1. Jinja2 SSTI 취약점 방어 및 텍스트 제한 설정을 통한 RCE 차단 및 메모리 장애(OOM) 방지
* **문제**
  - 투자 보고서 PDF 변환 모듈에서 사용자 입력값이 템플릿(Jinja2) 엔진에 직접 문자열 결합하여 실행되는 로직 분석
  - `{% raw %}{{7*7}}{% endraw %}` 페이로드 주입을 통한 SSTI(Server-Side Template Injection) 및 잠재적인 원격 코드 실행(RCE) 위험 식별
  - 악의적인 무제한 텍스트 렌더링 요청 시 발생할 수 있는 PDF 변환 서버 리소스 고갈(OOM) 임계점 파악
* **해결**
  - **입력값 보안 제어**: 사용자 유입 파라미터가 실행 구문으로 오동작하지 않도록 HTML 이스케이프(`escape()`) 필터 처리 전면 도입
  - **자원 임계치 차단**: 비정상 대형 PDF 파일 생성 요청에 대응하기 위해 입력 텍스트 문자열 길이를 **최대 25,000자**로 바인딩하는 유효성(Validation) 계층 추가
* **성과**
  - **보안 무결성**: 인젝션 취약 패턴을 사전 제거하여 RCE 원격 제어 취약점 해결 및 서비스 안정성 고도화
  - **안정성 강화**: 대량 파일 변환 과정에서의 자원 과부하 및 메모리 리스크 제거

#### 2. 비관적 락 & 상태 머신 기반 정합성 보장으로 누적 플랫폼 거래액 20억 원 돌파 기여
* **문제**
  - 금융/회계 도메인 특성상 대량 투자금 집행, 예치금 차감 및 채권 정산 로직의 데이터 무결성과 정합성 필수
  - 동시성 입출금 발생 시 데이터 데드락(Deadlock) 및 가입 잔액 부정합 방지 요구됨
* **해결**
  - **동시성 통제**: 원리금 이체 및 정산 트랜잭션 안정성을 확보하기 위해 DB 트랜잭션 격리수준 및 비관적 락(Pessimistic Lock / Select for Update) 설계 적용
  - **상태 관리 자동화**: 가상계좌 입금, 출금 대기, 이체 완료 등 결제 수명 주기를 투명하게 제어하기 위해 상태 머신(State Machine) 디자인 패턴을 금융 비즈니스 코어에 이식
* **성과**
  - **비즈니스 성장**: P2P 금융 시스템 코어를 안정적으로 구축하여 서비스 개시 이후 **누적 플랫폼 거래액 20억 원** 돌파에 직접적 기여
  - **무결성 유지**: 출시 이후 대사 일치율 100% 달성 및 금융 데이터 부정합 장애 0건 기록

#### 3. HTML Table 기반 인라인 CSS 템플릿 설계로 이메일 레이아웃 관련 고객 컴플레인 0건 단축
* **문제**
  - 주요 포털 이메일 서비스(Gmail, Naver, Daum 등)별로 CSS 표준 준수 스펙이 달라 수신된 보고서 레이아웃 손상 발생
  - 이메일 레이아웃 불일치로 인한 법인 고객 컴플레인 인입 및 고객지원 오버헤드 유발
* **해결**
  - **템플릿 재설계**: 표준 렌더링 편차를 우회하기 위해 외부 파일 로드 형식을 차단하고, 가장 넓은 하위 호환성을 가진 **HTML Table 기반 Inline CSS 구조**로 템플릿 레이아웃 전면 리팩토링
* **성과**
  - **고객 경험 제고**: 모든 국내외 이메일 수신 클라이언트 대상 레이아웃 호환성 100% 확보 및 렌더링 관련 컴플레인 접수 0건 달성
