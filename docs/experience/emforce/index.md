## 광고 데이터 플랫폼 운영 개선 및 성능·보안 고도화

| --------- | -------------------------------------------------------------- |
| **기간**    | 2024.12 ~ 2025.06 (7개월)                                      |
| **업무/프로젝트** | 사내 광고 플랫폼 운영 및 배치 안정화, API 기반 데이터 수집 전환, 서버 보안 취약점 진단 |

### 핵심 성과 및 의사결정 이력

#### 1. MemoryError 분석을 통한 Swap 메모리 확장 및 대용량 배치 수집 장애 재발률 0% 달성
* **문제**
  - AWS EC2 `t3a.small` (2GB RAM) 환경에서 대용량 광고 데이터 수집/병합 배치 실행 중 OOM `MemoryError` 및 배치 중단 수시 발생
  - 타 부서와 물리 인프라를 공유하는 구성 환경 제약으로 인해 인스턴스 사양 업그레이드(Scale-up) 불가능
* **해결**
  - **인프라 메모리 확장**: 무중단 OS 레벨 swap 공간(1GB ➔ 4GB)을 확장 구성하여 메모리 고갈 시 배치 비정상 종료 방어망 구축
  - **애플리케이션 메모리 최적화**: Pandas 연산 과정에서 불필요한 데이터프레임 복사본 생성을 방지하도록 인플레이스(in-place) 연산 도입 및 메모리 사용량이 큰 루프 직후 가비지 컬렉터(`gc.collect()`) 수동 호출
* **성과**
  - **장애 재발률 0%**: 동일 데이터 볼륨 조건에서 배치 프로세스 중단 없이 안정적이고 지속적인 완료율 100% 달성
  - **운영 비용 절감**: 하드웨어 업그레이드 없이 소프트웨어 및 OS 최적화만으로 시스템 안정성 확보

#### 2. API 기반 인증 방식 전환을 통한 광고 데이터 수집 시간 80% 단축 (3~5분 ➔ 30~45초)
* **문제**
  - Selenium 기반 브라우저 제어로 수집이 진행되어 데이터 렌더링 대기 등으로 수집 시간이 과도하게 발생(3~5분 소요)
  - 헤드리스 브라우저 구동으로 인한 심각한 CPU/메모리 부하 및 대상 웹 인터페이스 미세 변경 시 크롤러 상시 깨짐
* **해결**
  - **네트워크 프로토콜 역분석**: 브라우저 오버헤드를 완전히 걷어내기 위해 대상 광고 사이트 로그인 패킷을 캡처 및 정밀 분석
  - **인증 모듈 자체 구현**: **Keycloak 기반 OAuth2 인증 체계**의 토큰 발급/갱신 로직을 모방하여 순수 HTTP Request로만 데이터를 수집하는 구조로 전면 리팩토링
* **성과**
  - **처리 성능 향상**: 데이터 수집 시간 기존 **3~5분에서 30~45초로 약 80% 단축**
  - **시스템 부하 경감**: Selenium 프로세스가 점유하던 CPU/메모리 사용량을 제거하여 수집 서버 리소스 효율 극대화

#### 3. 모의 침투를 통한 SQL Injection 위협 선제 탐지 및 사내 보안 조치 표준 제안
* **문제**
  - AWS Lambda 서버리스 조회 API 코드 내 문자열 포맷팅(`f-string`)으로 사용자 입력값을 SQL에 결합하는 인젝션 보안 패턴 인지
  - 실제 데이터베이스 탈취 및 원격 실행 가능성과 비즈니스에 미치는 리스크에 대한 실증적 증명 요구됨
* **해결**
  - **모의 침투 검증**: 격리된 스테이징 RDS 환경에서 `SLEEP()` 구문을 결합한 Time-based SQL Injection 공격을 시뮬레이션하여 취약성 동작 실증 검증
  - **보안 표준 수립**: 라이브 DB 제어권 유출 리스크 검증 결과를 바탕으로 pymysql 파라미터 바인딩 적용 코드를 포함한 **"사내 개발 보안 취약점 조치 제안서"** 작성 및 전사 전파
* **성과**
  - **선제적 위험 제거**: 실 서버 배포 전에 치명적인 SQL Injection 위협을 진단하고 가이드를 반영하여 잠재적 유출 위험 차단
  - **보안 역량 강화**: 사내 개발 조직 내 안전한 SQL 쿼리 작성 표준 규칙 수립 및 개발 인식 제고

#### 4. Custom Test Runner 개발을 통한 한글 기반 검증 메트릭 가시화 및 디버깅 생산성 향상
* **문제**
  - 테스트 스위트 확장으로 테스트 케이스가 늘어나며 터미널에 영문 테스트 함수명만 출력되어 기능 변경 시 실패 원인 직관적 진단 불가
  - 테스트 코드 가독성이 떨어져 개발자들의 품질 검증 및 회귀 오류 디버깅 오버헤드 증가
* **해결**
  - **테스트 러너 확장**: Django 기본 `DiscoverRunner`를 상속 및 커스텀 구현하여 테스트 구동 런타임 수집 로직 확장
  - **가독성 패키징**: 리플렉션을 통해 테스트 타겟 클래스/메서드의 docstring(한글 설명문)을 가로채 터미널에 시인성 높게 출력하는 Custom Test Runner 빌드 및 테스트 환경 적용
* **성과**
  - **디버깅 속도 개선**: 수십 개의 실패 목록을 영문 함수명 역추적 대신 한글 설명 출력을 통해 원인 진단 속도 대폭 개선
  - **테스트 작성 동기화**: 직관적인 한글 피드백 제공으로 사내 개발팀의 테스트 코드 추가 작성 생산성 향상
